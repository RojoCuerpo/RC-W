Welcome

This is a test website

In the next slides you can see screenshots of the code that builds it. These feature a Flask server, HTML code that incorporates the Bootstrap framework, as well as a CSS style sheet. In combination, all of these files make this site work. More details below.

server.py

On the side of the server, we begin by importing the necessary packages, such as Flask, SQL and Datetime.
Then we create the model for the SQL database that will help us manage communications from the users.
Finally come the routes that will give access to the different pages on our website.
In the route '/contact' we use the db model along with the request method to catch the info and message from the user's communication. And finally we add it into the db.

base.html

This one starts by importing the Boostrap framework and the necessary JavaScript script to make the site look and work beautifully without too much effort.
Note the creation of several "blocks", this will aid us in adapting the content of the pages while keeping a consistent design throughout the site. These blocks work through a templating language called Jinja, which is embedded inside of Flask, so no need to import it in the server.
The footer can be outside the "blocks" since it will be the same in every page.

index.html

This is by far the busiest file on the site. It starts by extending the base.html template; doing this we can save many lines of code because the styles and script comes from there. Also, via the base.html, we are able to open the blocks to make the code more adaptable.
Inside these blocks we can then take advantage of Bootstrap to create a fully functional set of cards that render various images and their descriptions, as well as containing links to a display page that expands the view of each image.

about.html

This one consists on a card similar to the ones used in the index, but with a proportion that favors the image.

contact.html

In contact we use Bootstrap to create a simple, yet pretty-looking form that allows the user to write and send a message after filling the fields. The info is caught and sent to the server using a "POST" method, allowing the server to create a new entry on the database that manages communications. Check server.py
This page changes its display after the user has sent a message, making it more adaptable. This is done by implementing some Python code inside the html code, with the help of the Jinja templating capabilities.

display.html

Finally we have a display page that mutates to render any image you click on. This is, again, greatly aided by Jinja.
It starts by collecting and sending the parameters 'image' and 'ext' when an image is clicked, into the 'display' function on the server; then we adapt a couple of variables with these params to locate the right image and give the correct title to the display; and finally these new params are sent to the render_template.
Check the route 'display' on server.py to see the entire process and the solution we found to make it adaptable to each image and save some lines of code.

styles.css

Although Bootstrap is great, more often than not you'll need to twick its default styles to get the perfect look and feel to your project. To do so, we created a CSS styles sheet that keeps elements of the design (such as margins and paddings), as well as the look consistent throughout the site.
Special attention was placed on the fields of the Contact form, given the fact that the default shadows and focus of Bootstrap Forms are of a blue-ish hue.
